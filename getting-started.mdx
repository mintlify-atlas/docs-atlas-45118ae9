## Prerequisites

Before you begin, ensure you have the following installed on your system:

- **Node.js** 18.x or higher
- **pnpm** (preferred) or npm package manager
- A code editor (VS Code recommended)

<Note>
  This project uses `pnpm` as the package manager. While you can use `npm` or `yarn`, the documentation examples use `pnpm` commands.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    Get the source code from GitHub:
    
    ```bash
    git clone https://github.com/Doki2007/portfolio.git
    cd portfolio
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using pnpm:
    
    ```bash
    pnpm install
    ```
    
    This will install the following key dependencies:
    
    ```json
    {
      "astro": "^5.17.1",
      "tailwindcss": "^4.1.18",
      "@tailwindcss/vite": "^4.1.18",
      "sharp": "^0.34.5"
    }
    ```
  </Step>

  <Step title="Start the development server">
    Launch the local development server:
    
    ```bash
    pnpm dev
    ```
    
    The portfolio will be available at `http://localhost:4321`
    
    <Info>
      The dev server includes hot module replacement (HMR), so changes to your code will be reflected instantly in the browser.
    </Info>
  </Step>

  <Step title="Explore the portfolio">
    Open your browser and navigate to:
    
    - **Home**: `http://localhost:4321` - Main portfolio page
    - **Astro Dev Toolbar**: Visible in development mode for debugging
    
    You should see the complete portfolio with all sections:
    - Hero section with availability badge
    - About section with professional timeline
    - Projects showcase with featured work
    - Contact form with social links
  </Step>
</Steps>

## Available commands

All commands are run from the root of the project:

| Command | Action |
|---------|--------|
| `pnpm dev` | Starts local dev server at `localhost:4321` |
| `pnpm build` | Build production site to `./dist/` |
| `pnpm preview` | Preview your build locally before deploying |
| `pnpm astro` | Run Astro CLI commands |

### Development command

The development server provides:

```bash
pnpm dev
```

- Fast hot module replacement (HMR)
- TypeScript type checking
- Astro dev toolbar for debugging
- Accessible at `http://localhost:4321`

### Build command

Create an optimized production build:

```bash
pnpm build
```

This will:
1. Type-check all TypeScript files
2. Optimize and bundle assets
3. Generate static HTML pages
4. Optimize images with Sharp
5. Output to `./dist/` directory

<Info>
  The build output is fully static and can be deployed to any static hosting service like Vercel, Netlify, or Cloudflare Pages.
</Info>

### Preview command

Test the production build locally:

```bash
pnpm preview
```

This serves the `./dist/` folder to verify the build works correctly before deployment.

    plugins: [tailwindcss()],
  },

## Development workflow

<Steps>
  <Step title="Start the dev server">

  </Step>

  </Step>
</Steps>

## TypeScript configuration

The project uses Astro's strict TypeScript preset:

```json
{
  "extends": "astro/tsconfigs/strict",
  "include": [".astro/types.d.ts", "**/*"],
  "exclude": ["dist"]
}
```

This provides:
- Full type safety for Astro components
- IntelliSense in your editor
- Compile-time error checking
- Better refactoring support

<Note>
  The Navbar component includes TypeScript for the scroll-based navigation logic at `src/components/Navbar.astro:75`.
</Note>

## Troubleshooting

### Port already in use

If port 4321 is occupied, Astro will automatically use the next available port. Check the terminal output for the actual URL.

### Build errors

Ensure you're using Node.js 18.x or higher:

```bash
node --version
```

### Sharp installation issues

If Sharp fails to install, try:

```bash
pnpm install --force
```
