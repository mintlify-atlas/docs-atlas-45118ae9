## Key features

This portfolio demonstrates modern web development best practices:

### Performance-first architecture

Built with **Astro 5.x**, the portfolio leverages static site generation for near-instant page loads. The component-based architecture ensures optimal performance with minimal JavaScript shipped to the client.

```astro
// src/pages/index.astro - Main entry point
import Layout from "../layouts/Layout.astro";
import Navbar from "../components/Navbar.astro";
import Hero from "../components/Hero.astro";
import About from "../components/About.astro";
import Projects from "../components/Projects.astro";
import Contact from "../components/Contact.astro";
import Footer from "../components/Footer.astro";

<Layout>
  <Navbar />
  <Hero />
  <About />
  <Projects />
  <Contact />
  <Footer />
</Layout>
```

### Modern design system

**Tailwind CSS 4.x** powers a custom design system with:
- Custom color palette (ocean-dark theme with primary purple accent)
- Responsive typography using Inter and Playfair Display fonts
- Smooth animations and transitions
- Glass morphism effects and gradient backgrounds

## Tech stack

The portfolio is built with modern web technologies:

| Technology | Version | Purpose |
|------------|---------|----------|
| **Astro** | 5.17.1 | Static site generator and component framework |
| **Tailwind CSS** | 4.1.18 | Utility-first CSS framework |
| **TypeScript** | Latest | Type-safe JavaScript for better DX |
| **Sharp** | 0.34.5 | High-performance image optimization |
| **Vite** | Built-in | Fast build tool and dev server |

<Note>
  The portfolio uses Astro's built-in TypeScript support with strict configuration for type safety throughout the codebase.
</Note>

## Project philosophy

This portfolio demonstrates:

1. **Clean code practices** - Well-organized component structure with clear separation of concerns
2. **Performance optimization** - Static generation, optimized images, and minimal JavaScript
3. **Responsive design** - Mobile-first approach with breakpoints for all screen sizes
4. **Accessibility** - Semantic HTML, ARIA labels, and keyboard navigation support
5. **Developer experience** - Fast hot module replacement and TypeScript for better tooling
