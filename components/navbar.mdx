## Overview

The Navbar component provides site navigation with intelligent active link tracking based on scroll position. It features a sticky header design, smooth animations, and a TypeScript-powered Intersection Observer for automatically highlighting the current section.

## Component Code

```astro
<header class="fixed top-0 z-50 w-full border-b border-primary/10 bg-ocean-dark/90 backdrop-blur-md supports-backdrop-filter:bg-ocean-dark/60">
  <div class="mx-auto max-w-7xl px-6 py-5 flex items-center justify-between">
    <!-- Logo -->
    <!-- Navigation -->
    <!-- CTA Button -->
  </div>
</header>

<script lang="ts">
  // Navigation logic
</script>
```

<Note>
**Fixed Positioning:** The navbar uses `fixed top-0 z-50` to remain visible during scrolling with a high z-index to stay above other content.
</Note>

## Active Link Tracking Script

TypeScript-powered navigation state management:

<CodeGroup>
```typescript Initialization
document.addEventListener("DOMContentLoaded", () => {
  const sections = document.querySelectorAll("section");
  const navLinks = document.querySelectorAll(".nav-link");
  
  // Helper function and event listeners
});
```

```typescript Active Link Helper
const setActiveLink = (targetLink) => {
  // 1. Clear all links
  navLinks.forEach((link) => {
    link.classList.remove("text-primary");
    link.classList.add("text-slate-300");
    
    const line = link.querySelector(".nav-line");
    if (line) {
      line.classList.remove("w-2/3");
      line.classList.add("w-0");
    }
  });
  
  // 2. Activate target link
  if (targetLink) {
    targetLink.classList.remove("text-slate-300");
    targetLink.classList.add("text-primary");
    
    const activeLine = targetLink.querySelector(".nav-line");
    if (activeLine) {
      activeLine.classList.remove("w-0");
      activeLine.classList.add("w-2/3");
    }
  }
};
```
</CodeGroup>

### Click Event Handling

Manual navigation via clicks:

```typescript
navLinks.forEach((link) => {
  link.addEventListener("click", (e) => {
    setActiveLink(link);
  });
});
```

### Scroll Tracking with Intersection Observer

Automatic active link updates during scrolling:

<Tabs>
  <Tab title="Observer Configuration">
    ```typescript
    const observerOptions = {
      root: null,
      rootMargin: "0px",
      threshold: 0.3, // 30% visible
    };
    ```

    **Options Explained:**
    - `root: null` - Uses viewport as container
    - `rootMargin: "0px"` - No offset from viewport edges
    - `threshold: 0.3` - Triggers when 30% of section is visible
  </Tab>
  
  <Tab title="Observer Callback">
    ```typescript
    const observer = new IntersectionObserver((entries) => {
      entries.forEach((entry) => {
        if (entry.isIntersecting) {
          const id = entry.target.getAttribute("id");
          const activeLink = document.querySelector(`.nav-link[href="#${id}"]`);
          
          if (activeLink) {
            setActiveLink(activeLink);
          }
        }
      });
    }, observerOptions);
    ```

    **Flow:**
    1. Detects when section becomes 30% visible
    2. Gets section ID attribute
    3. Finds corresponding nav link
    4. Updates active state
  </Tab>
  
  <Tab title="Observer Attachment">
    ```typescript
    sections.forEach((section) => observer.observe(section));
    ```

    Observes all `<section>` elements on the page.
  </Tab>
</Tabs>

## Backdrop Blur Effect

The navbar uses modern CSS for glassmorphism:

```css
bg-ocean-dark/90: 90% opacity background
backdrop-blur-md: Blurs content behind navbar
supports-backdrop-filter:bg-ocean-dark/60: Feature query for browsers supporting backdrop-filter
```

<Note>
**Browser Support:** The `supports-backdrop-filter` prefix provides a fallback for browsers without backdrop-filter support.
</Note>

## Performance Considerations

- **Intersection Observer:** More efficient than scroll listeners
- **Debouncing:** Not needed due to Observer API efficiency
- **CSS Transitions:** Hardware-accelerated animations
- **Fixed Positioning:** Uses GPU compositing

<Note>
The navbar's scroll tracking works seamlessly with sections that have `id` attributes matching the `href` values in the navigation links (home, about, projects, contact).
</Note>
