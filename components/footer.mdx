## Overview

The Footer component provides a clean, minimalist bottom section for the portfolio site. It displays the brand name, a separator, and copyright text in a centered, responsive layout.

## Component Code

```astro
<footer class="bg-[#0b0e1b] py-8 border-t border-white/5">
  <div class="mx-auto max-w-7xl px-6 flex flex-col md:flex-row items-center justify-center gap-6">
    <div class="flex items-center gap-2">
      <span class="text-white font-bold text-lg">JAMIR.DEV</span>
      <span class="text-slate-600 mx-2">|</span>
      <p class="text-slate-500 text-sm">© 2026. Crafted with ocean vibes.</p>
    </div>
  </div>
</footer>
```

## Structure Breakdown

### Footer Container

The outer footer element sets the background and spacing:

```html
<footer class="bg-[#0b0e1b] py-8 border-t border-white/5">
  <!-- Content -->
</footer>
```

**Styling:**
- `bg-[#0b0e1b]` - Custom dark navy background (darker than main sections)
- `py-8` - 32px padding top and bottom
- `border-t border-white/5` - Subtle top border separator

### Content Wrapper

Centered container with responsive flex layout:

```html
<div class="mx-auto max-w-7xl px-6 flex flex-col md:flex-row items-center justify-center gap-6">
  <!-- Footer content -->
</div>
```

<Tabs>
  <Tab title="Mobile Layout">
    - `flex-col` - Stacks content vertically
    - `items-center` - Centers horizontally
    - `gap-6` - 24px spacing between elements
  </Tab>
  
  <Tab title="Desktop Layout">
    - `md:flex-row` - Horizontal layout on medium+ screens
    - `justify-center` - Centers content horizontally
    - Same gap maintained
  </Tab>
</Tabs>

## Content Elements

### Brand Name

Displays the site brand in bold white text:

```html
<span class="text-white font-bold text-lg">JAMIR.DEV</span>
```

**Properties:**
- White color for high contrast
- Bold weight for emphasis
- Large text size (18px)
- All caps for brand consistency

### Separator

Visual divider between brand and copyright:

```html
<span class="text-slate-600 mx-2">|</span>
```

**Features:**
- Vertical bar character
- Muted slate color
- Horizontal margin for spacing

### Copyright Text

Legal and creative credit:

```html
<p class="text-slate-500 text-sm">© 2026. Crafted with ocean vibes.</p>
```

**Styling:**
- Light slate color (more muted than separator)
- Smaller text size (14px)
- Friendly, personality-driven copy

<Note>
The copyright year (2026) and "ocean vibes" tagline add personality while maintaining professionalism.
</Note>

## Usage in Main Page

```astro src/pages/index.astro
---
import Footer from "../components/Footer.astro";
import Layout from "../layouts/Layout.astro";
---

<Layout>
  <Navbar />
  <Hero />
  <About />
  <Projects />
  <Contact />
  <Footer />
</Layout>
```

The Footer is placed last in the page layout, providing a natural conclusion to the content.

## Design Philosophy

### Minimalism

The footer embraces simplicity:
- No social links (already in Contact section)
- No navigation duplication
- No excessive information
- Clean, uncluttered design

### Color Hierarchy

<CodeGroup>
```css Text Colors
text-white: Brand name (highest contrast)
text-slate-600: Separator (medium contrast)
text-slate-500: Copyright (lower contrast)
```

```css Background
bg-[#0b0e1b]: Darker than main sections
border-white/5: Very subtle top border
```
</CodeGroup>

## Styling Consistency

### Theme Integration

The footer maintains consistency with the overall design:

- **Background:** Darker shade of ocean theme (`#0b0e1b`)
- **Borders:** Same low-opacity white as other sections
- **Typography:** Matches site-wide font stack
- **Spacing:** Uses consistent Tailwind spacing scale

### Border Treatment

```css
border-t border-white/5
```

This creates a subtle separation from the Contact section above without being visually heavy.

## Accessibility

<Note>
**Semantic HTML:** Uses proper `<footer>` element for screen readers and SEO.
</Note>

Accessibility features:
1. Semantic `<footer>` tag
2. Adequate color contrast (white on dark navy)
3. Readable font sizes
4. No interactive elements requiring focus states (in base version)

The Footer component is extremely lightweight:
