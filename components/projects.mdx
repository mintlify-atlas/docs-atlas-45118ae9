## Overview

The Projects component showcases featured work in an elegant card-based layout. Each project card includes an optimized image, technology tags, challenge highlights, and links to live demos and source code. The component uses Astro's Image component for performance optimization.

## Image Optimization

Astro's Image component provides automatic optimization:

<CodeGroup>
```astro Image Props
<Image
  src={cmsImage}
  alt="Descriptive alt text"
  class="w-full h-full object-cover object-top"
  loading="lazy"
  decoding="async"
  widths={[400, 800, 1200]}
  sizes="(max-width: 768px) 100vw, 50vw"
/>
```

```markdown Features
- **widths:** Generates responsive image sizes
- **sizes:** Defines image sizing at different viewports
- **loading="lazy":** Defers off-screen images
- **decoding="async":** Non-blocking image decode
- **object-cover:** Fills container while maintaining aspect ratio
- **object-top:** Aligns image from top
```
</CodeGroup>

## Styling Highlights

### Glass Card Effect

The cards use a combination of techniques:
- Semi-transparent backgrounds
- Border with low opacity
- Shadow effects for depth
- Backdrop blur (via `glass-card` class)

### Hover Animations

1. **Card Lift:** Entire card translates upward
2. **Image Zoom:** Image scales and rotates slightly
3. **Border Glow:** Border opacity increases
4. **Shadow Enhancement:** Shadow becomes more prominent

### Color Coding

Different technologies use distinct color schemes:
- **Astro:** Primary/lavender (`bg-primary/20`)
- **Angular:** Red (`bg-red-600/20`)
- **TypeScript:** Blue (`bg-blue-600/20`)
- **Neutral:** White/dark navy for generic tags

<Note>
All animations use `transition-all` with specified durations for smooth, cohesive interactions.
</Note>
